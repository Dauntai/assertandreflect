# Assert and Reflect

A static website about testing philosophy and quality engineering. *Testing is about creating confidence, not just finding defects.*


## Overview

Assert and Reflect offers:

- **Practical Guides** — Hands-on advice for designing tests and automation
- **Architecture & Strategy** — Frameworks for testing at scale and risk modeling
- **Reflections on Quality** — Thoughtful essays on what testing means and why it matters

The site is fully static: HTML, CSS, and minimal vanilla JavaScript. No backend required. Deploy anywhere that serves static files (e.g., Hostinger, Netlify, GitHub Pages).

## Prerequisites

- **Node.js** 16 or later (for the build script)

## Local Build

1. Install dependencies:

   ```bash
   npm install
   ```

2. Build the site (converts Markdown to HTML, generates `posts.json`, `sitemap.xml`, `robots.txt`):

   ```bash
   npm run build
   ```

   For production deployment to a custom domain, set the base URL before building:
   ```bash
   SITE_URL=https://yourdomain.com npm run build
   ```

3. Serve locally (optional, for preview):

   ```bash
   npx serve .
   ```

   Or use any static file server. Opening `index.html` directly may not work for post loading because of how `fetch` handles file URLs.

## Deploy to Hostinger

1. Run the build: `npm run build`
2. Upload the following to your Hostinger `public_html` folder:
   - `index.html`
   - `about.html`
   - `guides/` (entire folder, includes category index + post HTML files)
   - `strategy/` (entire folder)
   - `reflections/` (entire folder)
   - `assets/` (entire folder)
   - `posts.json` (generated by build)
   - `sitemap.xml` (generated by build)
   - `feed.xml` (generated by build, RSS feed)
   - `robots.txt` (generated by build)

   You can upload everything *except* `content/`, `build.js`, `package.json`, `package-lock.json`, and `node_modules/`.

3. Your site will be live at your domain root.

## Adding New Posts

1. Create a new `.md` file in the appropriate folder:
   - `content/guides/` — Practical guides
   - `content/strategy/` — Architecture and strategy
   - `content/reflections/` — Reflections on quality

2. Add YAML frontmatter at the top:

   ```yaml
   ---
   title: Your Post Title
   date: 2025-02-19
   category: guides   # guides | strategy | reflections
   excerpt: A short description for the post listing.
   slug: optional-custom-slug   # auto-generated from filename or title if omitted
   ---
   ```

3. Write your content in Markdown. Use one H1 (the title) and multiple `##` (H2) / `###` (H3) for SEO. The build will warn if headings are missing.

4. **Images** (optional): Place in `assets/images/posts/<post-slug>/` (e.g. `assets/images/posts/confidence-vs-defect-counting/diagram.svg`). Use `![Descriptive alt text](/assets/images/posts/<slug>/<filename>)` — alt text is required for accessibility. See `editorial-style.md` for hero images and examples.

5. Run the build: `npm run build`

6. Redeploy the updated category folders and `posts.json`.

## SEO

The site includes production-grade SEO:

- **Meta tags** — Unique titles, descriptions, canonical URLs, Open Graph, and Twitter cards on every page
- **Structured data (JSON-LD)** — BlogPosting schema for posts, Organization and Person schemas site-wide
- **Sitemap** — `sitemap.xml` generated automatically during build
- **RSS feed** — `feed.xml` for readers and aggregators
- **robots.txt** — Allows all crawlers, references sitemap
- **Semantic HTML** — `<article>`, `<section>`, `<nav>`, `<header>`, `<footer>` with proper heading hierarchy

Set `SITE_URL` before build for your production domain (e.g. `SITE_URL=https://assertandreflect.com npm run build`).
